# 公交广告管理系统 - 产品设计方案

## 一、项目概述

### 1.1 系统定位
公交广告管理系统是一套面向广告管理单位的企业级管理平台，旨在实现广告资源管理、订单管理、上刊执行、巡检监控、财务结算、数据分析的全流程数字化管理。

### 1.2 核心用户角色
- **广告管理单位管理员**:系统管理、资源管理、合作方管理
- **销售人员**:客户拓展、报价、合同签订、订单管理
- **审核人员**:广告内容审核、资质审核、合规检查
- **财务人员**:费用结算、发票管理、财务报表
- **公司管理层**:经营数据查看、流程审批、决策支持
- **上刊执行人员**:通过小程序执行上刊、巡检、下刊、修补作业
- **广告发布公司(合作方)**:订单提交、素材上传、进度查询

### 1.3 业务价值
- 资源数字化管理,提升资源利用率
- 全流程在线化,减少人工成本
- 移动化作业,提高执行效率
- 数据可视化,支持经营决策
- 合规化管理,降低运营风险

### 2.2 技术栈建议
- **前端**: Vue3 + Element Plus + TypeScript
- **小程序**: UniApp / 原生微信小程序
- **后端**: Spring Boot + MyBatis Plus
- **数据库**: MySQL 8.0 + Redis
- **存储**: 阿里云OSS / MinIO
- **部署**: Docker + Kubernetes

**功能点**:
- 资源批量导入(Excel)
- 资源分类管理
- 资源状态追踪

- 资源库存统计
- 资源占用率分析

#### 3.1.2 线路管理

- 投放周期(开始日期-结束日期)
- 合同金额
- 付款方式
- 订单状态
- 合同附件
- 广告素材
- 创建时间

**订单状态**:
- 待审核
- 审核通过
- 审核驳回
- 待上刊
- 已上刊
- 巡检异常
- 已下刊
- 已完成
- 已取消

**功能点**:
- 订单快速创建
- 资源冲突检测
- 自动排期推荐
- 订单变更管理
- 订单延期处理
- 订单历史追溯

#### 3.2.2 合同管理
**功能描述**:广告合同的数字化管理

**功能点**:

## 四、小程序端功能设计

### 4.1 小程序角色定位
主要服务于上刊执行人员的现场作业

### 4.2 核心功能

#### 4.2.1 工作台
**功能点**:
- 今日任务统计
- 待办任务列表
- 快速操作入口
- 消息通知
- 个人信息

#### 4.2.2 上刊作业
**操作流程**:
1. 接收上刊任务
2. 查看任务详情(资源位置、素材信息)
3. 导航到现场(地图导航)
4. 上刊前拍照(记录原状)
5. 执行上刊
6. 上刊后拍照(多角度)
7. 填写上刊信息(实际尺寸、安装方式)
8. 提交完成

**必填信息**:
- 完成时间
- 现场照片(至少3张)
- 画面质量(完好/轻微瑕疵/有瑕疵)
- 特殊情况说明

#### 4.2.3 巡检作业
**操作流程**:
1. 接收巡检任务
2. 扫描二维码(识别资源)
3. 拍照记录(自动打水印)
4. 巡检结果选择(正常/异常)
5. 异常情况描述
6. 提交巡检记录

**快捷巡检**:
- 批量扫码巡检
- 一键拍照
- 语音输入问题
- 离线作业(网络不佳时)

#### 4.2.4 下刊作业
**操作流程**:
1. 接收下刊任务
2. 查看下刊资源
3. 下刊前拍照
4. 执行下刊
5. 下刊后拍照(验证完全移除)
6. 素材回收登记
7. 提交完成

#### 4.2.5 修补作业
**操作流程**:
1. 接收修补工单
2. 查看问题描述
3. 准备修补物料
4. 修补前拍照
5. 执行修补
6. 修补后拍照
7. 质量确认
8. 提交完成

#### 4.2.6 问题上报

## 六、界面设计规范

### 6.1 设计风格
- **风格定位**:企业级中后台系统
- **色彩方案**:
  - 主色:#1890FF (专业蓝)
  - 成功:#52C41A
  - 警告:#FAAD14
  - 错误:#F5222D
  - 文字主色:#262626
  - 文字次要:#8C8C8C
  - 边框:#D9D9D9
  - 背景:#F0F2F5

- **字体规范**:
  - 标题:16px-20px,加粗
  - 正文:14px,常规
  - 辅助信息:12px
  - 字体:PingFang SC, Microsoft YaHei, sans-serif

- **间距规范**:
  - 页面边距:24px
  - 组件间距:16px
  - 内容间距:8px

- **圆角规范**:
  - 卡片:4px
  - 按钮:4px
  - 输入框:4px

### 6.2 组件规范
- **按钮**:
  - 主按钮:蓝色背景,用于主要操作
  - 次要按钮:白色背景,灰色边框
  - 危险按钮:红色,用于删除操作
  - 文字按钮:无背景,用于次要链接

- **表格**:
  - 表头背景:#FAFAFA
  - 斑马纹:奇数行白色,偶数行#FAFAFA
  - 行高:54px
  - 操作列固定在右侧

- **表单**:
  - 标签右对齐
  - 必填项红色星号标识
  - 输入框宽度:100%(响应式)
  - 错误提示红色文字在输入框下方

- **状态标签**:
  - 成功:绿色
  - 进行中:蓝色
  - 警告:橙色
  - 失败/异常:红色
  - 禁用:灰色

### 6.3 交互规范
- **反馈**:
  - 操作成功:顶部绿色提示,2秒自动消失
  - 操作失败:顶部红色提示,4秒自动消失
  - 确认操作:模态对话框确认

- **加载**:
  - 页面加载:骨架屏
  - 按钮加载:加载图标+禁用状态
  - 列表加载:底部加载更多

- **空状态**:
  - 暂无数据提示
  - 引导性操作按钮
